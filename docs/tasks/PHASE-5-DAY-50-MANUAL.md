# Day 50 — Manual: semantic search UI

`GET /search/semantic` needs the feature flag, an embedding provider, and at
least one **active** property whose embedding has been generated.

## Enable

In the repo root `.env` (and restart the API):
```
ENABLE_SEMANTIC_SEARCH=true
OPENAI_API_KEY=sk-...            # text-embedding-3-small (1536 dims)
```

Embeddings are generated by the BullMQ worker when a property is published
active (ADR 007). Run the worker:
```bash
pnpm --filter @propai/api worker:dev
```
Create/publish a few active listings and wait for the embedding jobs to finish.

## Steps

1. Go to `localhost:3001/search`. The empty state shows example queries.
2. Run "Quiet area near parks, pet friendly, under $500k". Results render with a
   "NN% match" chip (hybrid `relevanceScore`, ADR 008).
3. Toggle **Sort** → Price low→high / Newest and confirm the order changes.

## Flag off / no provider

With `ENABLE_SEMANTIC_SEARCH=false` (or no API key) the endpoint returns `503`
and the page shows "AI search is warming up → Browse all listings" instead of an
error — the marketplace stays usable.
